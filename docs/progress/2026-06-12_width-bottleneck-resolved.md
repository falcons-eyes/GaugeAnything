# 2026-06-12 (3부) — 폭 병목 해체: "mask=WHERE, signal=WIDTH"

질문(사용자): "마스크 정제가 마지막 방법이면, 안 되면 제품 불가인가? 물리적으로 못 푸는
문제인지 냉정하게 판단하라. 우회 말고, 검토 안 했던 breakthrough를 찾아라."

## 단계 0 — 가정의 해체

"폭 측정엔 정확한 마스크 경계가 필요하다"는 가정 자체를 의심. 반례가 이미 우리 데이터에:
저자 DLM 11.1μm(같은 신호), 우리 rung1 35μm(올바른 위치 + 단순 규칙). **정보는 이미지에
살아있고, 죽는 지점은 binary mask 변환.** → 물리 분석 + 방법 공간 A/B/C/D 전수
([WIDTH_BOTTLENECK_ANALYSIS.md](../WIDTH_BOTTLENECK_ANALYSIS.md)).

핵심 재정의: **마스크 = WHERE(중심선), 폭 = 원신호에서 직접(HOW WIDE).**
마스크 경계 정확도 요구가 사라진다.

## 단계 1 — 추정기 검증 (oracle 위치)

| 추정기 | table test (19k, 그룹분할) | 이미지 추출 프로파일 |
|---|---:|---:|
| **1D CNN** (신규 — krkCMd 19k 학습, CC BY 클린) | **17.9μm** | **26.2μm** |
| minrun5 (기존 규칙) | 31.3 | 57.9 |
| EW equivalent-width (신규 가설) | — | 128.5 **기각** (텍스처가 배경 오염 — 정직 기록) |
| 앵커: 저자 DLM | 11.1 | — |

→ **신호 추정은 해결.** 우리 1D CNN이 기존 최선(cal 25.9)을 넘고 DLM의 1.6× 거리.

## 단계 2 — 위치 진단·수정 (3회 반복)

| 회차 | 위치 방법 | 결과 | 진단 |
|---|---|---|---|
| 1 | 전역 SAM3 ds2 스켈레톤 | 오차 중앙 **1,090px** | 내부 리사이즈로 크랙 ~7px화 → 다른 어두운 구조(시편 가장자리) 오인 |
| 2 | + snap-to-valley(±80px) | 변화 없음 | 행 오프셋이 아니라 구조 오인이 원인 |
| 3 | **타일 추론 + 최암·최장 성분** | 성공률 46~66%, 성공 시 중앙 **17px** | 타일에선 크랙이 ~29px로 보임 |

## 단계 3 — 최종 판정 (end-to-end, 물리 GT)

| 조건 | n | 1D CNN MAE | 중앙 |
|---|---:|---:|---:|
| **위치 성공 시** | 113 | **39.9μm** | **23.2μm** |
| 위치 실패 시 | 53 | 186μm | 191μm |
| (binary mask 폭, 이전 최선) | — | 144~186μm | — |

**객관 판정**:
1. **물리적 불가 아님 — 확정.** 위치만 맞으면 promptable 단안 RGB로 23~40μm
   (binary mask 대비 4~6×, 성공 기준 60μm 충족).
2. 잔여 항목은 **위치 커버리지 하나** (46~66%) — 학습 난제가 아니라 고전 CV 엔지니어링
   (능선 추적, hysteresis, 프로파일 신뢰 게이트). 또한 이 도메인(스캐너 시편) 특수성이 큼 —
   제품 도메인(현장 사진)은 CrackSeg9k에서 SAM3 위치가 이미 작동.
3. **제품 문법 확정**: 위치 게이트 통과 지점만 폭 보고 + 커버리지·σ 명시, 실패 지점은
   '측정 불가' 정직 표기. 100% 지점 측정을 약속하지 않는 것이 올바른 계측기다.
4. 실패한 가설도 기록: EW(PSF 불변)는 텍스처 배경에서 기각. 깨끗한 표면(금속)에선 재평가 여지.
