# 폭 측정 병목의 물리 분석과 방법 공간 전수 검토

날짜: 2026-06-12 · 동기: "마스크 정제가 안 되면 제품 불가인가?"에 대한 냉정한 답.

## 0. 결론 먼저 (객관 판정)

**물리적으로 풀 수 없는 문제가 아니다. 우리가 문제를 잘못 분해하고 있었다.**

증거 (전부 우리 실측):
| 측정 경로 | krkCMd 물리 GT MAE | 시사점 |
|---|---:|---|
| 저자 DLM (1D 학습, 원신호) | **11.1μm (≈2.8px)** | 정보가 이미지에 존재 — sub-pixel까지 |
| 우리 rung1 (올바른 위치의 원신호 + 단순 규칙) | 35μm | 위치만 맞으면 단순 규칙도 근접 |
| 우리 rung2 (SAM3 binary mask 폭) | 144~264μm | **정보가 죽는 지점 = binary mask 변환** |

→ 병목은 "마스크 품질"이 아니라 **"마스크 기하에서 폭을 읽는다"는 설계 가정**.
마스크의 역할을 **WHERE(중심선 위치)로 한정**하고, **HOW WIDE는 원본 이미지 신호에서
직접 추정**하면 마스크 경계 정확도 요구가 사라진다. SAM3의 crack 마스크는 경계는 부정확해도
중심선 통과는 잘한다(mIoU 0.44 = 위치는 맞음).

## 1. 물리: 단일 RGB에서 크랙 폭은 어디까지 가능한가

크랙 = 어두운 슬릿. 이미지화 과정: 실폭 w → PSF 블러 ⊗ → 샘플링 → 노이즈.
- **블러는 정보를 섞지만 없애지 않는다**: 컨볼루션은 적분(플럭스)을 보존 →
  intensity deficit의 적분은 PSF에 1차 불변. (천문 분광학의 **equivalent width** 원리)
- **경계 천이는 sub-pixel 정보를 담는다**: edge spread function을 erf/sigmoid로 적합하면
  경계를 1px 이하로 위치 추정 가능 (MTF 측정의 표준 기법).
- **학습 가능성의 실증**: DLM 11.1μm = 2.8px — 같은 센서 신호에서 이미 달성됨.

**진짜 물리 한계** (정직):
- w ≪ PSF이고 대비가 노이즈에 묻히면 분리 불가 — 그러나 deficit 적분(EW)은 w·대비의
  곱을 보존하므로, 대비를 모델링/보정하면 w ~ 0.5px까지도 추정 가능 (불확실성 증가).
- 제품 기준점: 점검 임계 0.2~0.3mm는 일반 촬영거리(0.1~0.3mm/px)에서 1~3px —
  **sub-pixel 기법이 필수인 영역이며, 불가능 영역은 아니다.** 단 σ를 함께 보고해야 한다
  (계측기 문법). 0.05mm 이하 + 원거리 촬영은 단일 프레임으론 한계 → C 트랙.

## 2. 방법 공간 전수 (검토 여부 표기)

### A. 신호 기반 폭 추정 — "mask for WHERE, signal for HOW WIDE" ★ 주공
| 방법 | 원리 | 상태 |
|---|---|---|
| A1 수직 프로파일 + 규칙 | 스켈레톤 법선 방향 원신호 추출 → minrun/half-depth | rung1로 간접 검증(35μm), **E-mm-3c에서 SAM3 위치로 시험** |
| A2 **Equivalent Width** | deficit 적분/깊이 — PSF 1차 불변 | **미검토였음 — 신규**, E-mm-3c |
| A3 ESF sub-pixel 경계 | 양 경계 erf 적합 → 변곡점 거리 | 미검토였음 — 신규 |
| A4 1D 학습 헤드 (DLM 재현) | 19k 프로파일(CC BY)로 1D CNN 학습 | **미검토였음 — 신규**, E-mm-3c. 라이선스 클린 학습데이터가 이미 손에 있음 |

### B. 마스크 자체 개선 — 보조
| 방법 | 상태 |
|---|---|
| B1 로짓 iso-level (θ*) | 검증: −30~40% (M2v2-a) — 천장 확인됨 |
| B2 소프트 로짓 sub-pixel iso-contour | 부분(soft.py 보유) — 288² 로짓 업샘플 한계 |
| B3 SAM3 thin fine-tune (LoRA) | 미실행 — 비용 큼, A가 성공하면 우선순위 하락 |
| B4 full-res 타일 추론 | 미실행 — A의 위치 정밀도 보강용으론 유효 |

### C. 캡처 측 (제품 설계 자유도 — 단일 프레임 한계의 우회가 아니라 확장)
| 방법 | 비고 |
|---|---|
| C1 버스트/비디오 sub-pixel SR | 스마트폰 2초 영상 요구 — 제품 스펙으로 흡수 가능 |
| C2 플래시 on/off 쌍 | 그림자/조명 분리 |
| C3 접사+기지 거리 | FalconEyes 운영 가이드 |

### D. 계측 문법 (방법이 아니라 의무)
- 폭<3px 영역은 점추정 대신 **구간 보고** (Var≈Σα(1−α) 보유) — 이미 설계에 있음.

## 3. 냉정한 실패 조건 (이래도 안 되면 인정해야 하는 것)
1. E-mm-3c에서 SAM3-위치 신호 추정이 rung1(35μm)에 못 미치고 100μm+에 머무름
   → 위치(스켈레톤)가 병목이라는 뜻 → B4/B3로 위치만 보강 후 재시험.
2. A4(1D 학습)가 table test에서 DLM(11μm) 대비 2× 이상 뒤처짐 → 우리 모델링 역량 문제,
   krknet 구조 추적 필요.
3. 실패 모드가 "위치도 신호도 OK인데 GT와 계통 편차" → 스케일 체인 재점검.
어느 경우도 "물리적 불가"가 아니라 특정 모듈의 개선 과제로 환원된다. **물리적 불가
판정이 정당해지는 유일한 영역**: w·대비가 노이즈 바닥 이하 (예: 원거리+0.05mm) —
이 영역은 σ 보고 + C 트랙으로 제품 스펙에서 명시 제외하는 것이 옳다.
